# Bug Report — AgentsFieldView

## Sintomas observados
- Jogadores travados (não se movem ou ficam parados)
- Bola tremendo (oscilação rápida sem progressão)
- Possível loop/ciclo impedindo movimento real

## Suspeitos principais

1. **MovementBridge velocidade muito baixa**
   - `TICK_SEC = 1/24` × `SPEED_WALK_BASE=7.05` / `FIELD_LENGTH_M=105` × 100 = ~0.28 normalized units/tick
   - Pode ser pequeno demais para mover visivelmente a 24fps

2. **stepSimulator chamado sem dt real**
   - AgentsFieldView chama `stepSimulator` a cada frame do rAF
   - Mas o bridge usa `TICK_SEC = 1/24` fixo — se o rAF rodar mais rápido/lento, dessincroniza

3. **Ball carrier loop**
   - `closestToBall` sempre retorna o mesmo agente
   - Agente decide PROGRESS → move em direção ao gol
   - Mas `newBallPos` é atualizado DEPOIS do bridge → bola e carrier dessincronizam
   - Próximo tick: carrier não é mais o mais próximo → posse muda → bola reseta → loop

4. **PlayerAgent ainda move currentPosition internamente**
   - `tickAgent` ainda tem `stepToward` no corpo
   - Bridge também move → double movement ou conflito

- `agents/core/PlayerAgent.ts` — verificar se stepToward ainda está ativo
- `agents/sim/TeamSimulator.ts` — ordem: tick → bridge → ball update
- `agents/integration/MovementBridge.ts` — velocidade real em normalized units
- `src/pages/AgentsFieldView.tsx` — rAF loop e frequência de stepSimulator
