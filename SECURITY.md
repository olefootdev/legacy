## Security Policy

### Reporting a Vulnerability

If you discover a security vulnerability in Olefoot, please email `exp@olefoot.com` with:

1. **Description** — What is the vulnerability?
2. **Location** — Where in the code does it occur?
3. **Impact** — How could it be exploited?
4. **Proof of Concept** — Steps to reproduce (if applicable)

We will:
- Acknowledge receipt within 24 hours
- Work with you to understand and fix the issue
- Give you credit (unless you prefer anonymity)
- Release a patch as soon as possible

### Out of Scope

- Social engineering attacks
- Denial of service attacks
- Physical security issues
- Issues in third-party services or dependencies
